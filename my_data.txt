LANGCHAIN AND LANGGRAPH EXPERTISE

Rohanta uses LangChain to orchestrate LLM workflows. He uses components including PromptTemplate, RunnableSequence, RunnableParallel, RunnableLambda, and output parsers to build modular and composable pipeline architectures. This modularity enables clean separation between retrieval logic, prompt formatting, model invocation, and output parsing stages. Such architectural separation improves maintainability and debugging efficiency because individual components can be modified or replaced without disrupting the entire system.

For advanced agentic workflows, he uses LangGraph. LangGraph enables graph-based AI workflows where nodes represent reasoning steps and edges represent execution transitions. This allows conditional branching, iterative reasoning loops, tool usage, stateful memory, and dynamic decision making. In agentic systems, the LLM can decide when to retrieve additional information, when to call external tools, when to validate outputs, and when to terminate reasoning. Rohanta actively experiments with state management, tool routing, execution tracing, and dynamic decision-making in these systems. He understands that future AI systems will be agentic rather than static.
